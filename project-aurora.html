Project Aurora

대형 스크린과 모바일 랜딩이 동시에 쓰이는 브랜드 쇼케이스. 아주 큰 이미지와 짧은 설명, 긴 리서치 문단이 반복되며 스크롤로 밀도를 쌓는 구조입니다.

Lead

민지 오

Stack

React, WebGL, Motion, Figma

첫 화면은 설명보다 분위기.

메인 이미지는 Cargo 레퍼런스처럼 크게 밀어 올리되, shadcn 카드 구조를 차용해 캡션과 메타 정보를 얇은 보더 카드 안에 담았습니다. 그래서 화면은 여전히 실험적으로 보이지만, 정보는 더 빠르게 읽힙니다.

특히 모바일에서는 이미지 비중이 너무 무거워지지 않도록 캡션을 이미지 아래로 분리하고, 스크롤이 진행될수록 설명 블록이 더 자주 등장하도록 리듬을 재조정했습니다.

정보 위계는 작지만 단단하게.

본문은 긴 글을 견디는 페이지여야 해서, 모든 섹션에 번호, 짧은 서브 타이틀, 본문, 메타 조합을 반복 배치했습니다. 디테일은 단순하지만 같은 구조가 여러 번 나오면서 프로젝트의 설계 논리를 계속 상기시킵니다.

또한 좌우 배치를 번갈아 사용해 사용자가 긴 페이지를 넘기고 있다는 느낌을 받지 않도록 했습니다. 정적이지만 단조롭지 않은 흐름이 핵심입니다.

Long Form Notes

스크롤을 많이 해야 하는 이유를 만드는 본문.

이 페이지는 단순히 긴 것이 아니라, 읽을수록 새로운 층위가 드러나도록 짜였습니다. 첫 섹션이 전체 방향을 잡아주고, 두 번째 섹션은 시스템을 설명하며, 세 번째 이후부터는 실제 화면에서 어떤 조정이 이뤄졌는지 세세하게 푸는 방식입니다. 그래서 방문자는 프로젝트를 한 번에 소비하지 않고, 스크롤을 통해 천천히 해석하게 됩니다.

레이아웃은 크게 세 가지 규칙을 따릅니다. 하나, 큰 이미지는 언제나 문단보다 먼저 나온다. 둘, 설명 카드의 최대 폭은 제한해 읽기 피로를 줄인다. 셋, 중요한 문장은 주변 여백으로 강조한다. 이 세 규칙만으로도 긴 문서 안에 안정적인 리듬이 생깁니다.

시각적으로는 둥근 모서리, 얇은 보더, 반투명 배경, 묵직한 타이포를 결합해 shadcn식 제품 페이지 감각을 유지했습니다. 하지만 지나치게 앱 같은 화면이 되지 않도록 배경에는 부드러운 그리드와 빛 번짐을 남겨 두었습니다. 이 균형 덕분에 포트폴리오이면서도 전시형 랜딩 같은 기분이 남습니다.

마지막으로, 본문이 길수록 요약이 중요합니다. 그래서 각 장면 끝에는 짧은 결론을 붙여 사용자가 어디까지 읽었는지 놓치지 않게 했습니다. 이 패턴은 다른 프로젝트 페이지로 확장해도 유지할 수 있는 공통 언어가 됩니다.

결론: 비슷하지만 더 현재적인 포트폴리오.

참고 사이트의 미니멀한 구조와 긴 스크롤 경험을 유지하되, 카드 시스템과 메타 스타일을 shadcn 감각으로 재정리했습니다. 덕분에 너무 똑같지 않으면서도 같은 계열의 인상을 유지합니다.
