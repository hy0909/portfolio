프로젝트 1

마트 상품 결품 알림 및 대시보드 서비스

초기 AI 스타트업에서 B2B 서비스를 기획 및 디자인하고 있습니다. 복잡한 정보 구조와 사용자 플로우를 쉽게 이해하고 사용할 수 있는 경험으로 만듭니다.

카테고리

대형마트, Vision AI

프로젝트 기간

2025. 7. 10 ~ 7. 31

팀

PM 1, 프로덕트 디자이너 1, AI 엔지니어 1, 프론트엔드 개발자 1, 백엔드 개발자 1

문제 상황

매출 손실 + 운영 비효율 + 의사결정 지연이 동시에 발생

마트 내 결품(Out-of-Stock)은 실시간으로 감지되지 않았고, 현장 직원의 순회 점검에 의존하고 있었습니다.
이로 인해 아래와 같은 문제가 발생했습니다.

· 결품 발생 시점을 알 수 없어 대응 지연 · 매대 상태 확인을 위해 불필요한 동선 발생 (비효율) · 결품 → 충진 → 판매까지의 흐름이 데이터로 남지 않음

문제 정의

이 프로젝트의 핵심 문제는 단순히 "결품을 알려주는 것"이 아니라, 결품 발생 이후의 전체 운영 프로세스가 측정되지 않고, 연결되지 않는 구조에 있었습니다.

· 언제 / 어디서 / 얼마나 오래 결품이 발생했는지 알 수 없음 · 결품 이후 현황이 추적되지 않음 · 병목(진열, 재고, 발주)이 어디서 발생하는지 파악 불가

문제 해결

1 결품 → 행동 → 결과, 단계별 데이터 구조 설계

각 단계별 이벤트의 시간을 측정 가능하도록 구조 설계하여, 운영 프로세스를 데이터로 변환하고자 했습니다.

2 현장 직원용: 행동을 유도하는 모바일 알림 설계

현장 직원이 사용할 모바일 알림 뷰는 '언제 / 어디 / 어떤 매대인지 한눈에 파악 가능하게 구성'했습니다. 불필요한 정보를 제거하여 '알림 → 확인 → 처리완료' 까지 이어지는 플로우를 최소화하여 설계했습니다.

· 단순 확인이 아닌 행동 기록(충진 여부 등)을 남길 수 있도록 설계 · 현장 행동의 정량화

3 관리자용: 인사이트 중심 PC 대시보드 설계

결품 데이터 모니터링 뿐만 아니라, 시간대 / 구역별 결품 발생 패턴을 파악할 수 있도록 차트 형태로 시각화하였습니다. 또한, 결품 → 충진까지 소요 시간을 기입하여 병목 구간이 있는 장소를 빠르게 확인할 수 있도록 했습니다.

결과

결품 관리의 정량화 달성

기존엔 현장 직원/마트 관리자의 감에 의존하거나, 결제 데이터만으로 의사결정할 수 있었지만, 현재는 결품 데이터 기반 운영으로 전환되어 결품 대응 시간 및 프로세스 개선 가능해졌습니다.
