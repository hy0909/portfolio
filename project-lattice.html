문제 상황

데이터는 참고 수준, 운영은 감에 좌우

저가 커피 매장은 빠른 회전율과 가격 경쟁력을 중심으로 운영되지만, 실제 매장 운영은 여전히 점주의 경험과 직관에 크게 의존하고 있었습니다. PoS를 통해 매출과 판매 데이터는 지속적으로 축적되고 있었지만, 고객 특성이나 시간대, 외부 환경과 같은 맥락 정보와 연결되지 않아 데이터가 실질적인 의사결정에 활용되지 못하는 한계가 있었습니다.

문제 정의

데이터가 '정책'에 머물고, '행동'으로 이어지지 않는 문제

기존 매장 데이터는 매출, 판매량, 주문 건수 등 개별 수치 중심으로 제공되고 있어, 현재 상황을 해석하거나 이후 행동으로 연결하기 어려운 구조였습니다. 어떤 고객이 주요 매출을 만드는지, 어떤 시간대에 어떤 메뉴가 잘 팔리는지, 특정 상황에서 어떤 전략을 적용해야 하는지에 대한 정보가 부재했습니다. 결국 문제의 본질은 데이터의 부재가 아니라, 데이터가 의사결정과 행동으로 연결되지 않는 구조에 있었습니다.

문제 해결 1

결품 → 행동 → 결과, 단계별 현황 기록

각 단계 발생 시간을 측정 가능하도록 구조를 설계하여, 운영 프로세스를 데이터로 변환하고자 했습니다.

문제 해결 2

현장 직원용: 행동을 유도하는 모바일 알림 설계

현장 직원이 사용할 모바일 알림 뷰는 '언제, 어디, 어떤 매대'인지 한눈에 파악 가능하게 구성했습니다. 불필요한 정보를 제거하여 '알림 → 확인 → 처리완료'까지 이어지는 플로우를 최소화하여 설계했습니다.

또한 '재고없음/처리완료/오류제보' 버튼을 추가하여 알림 후 담당 직원이 어떤 대응을 했는지 현장 직원들끼리 현황을 공유할 수 있도록 했습니다.

· 단순 결품 확인이 아닌 충진 여부 등 행동 기록을 남길 수 있도록 설계 · 현장 행동의 정량화

문제 해결 3

관리자용: 인사이트 중심 PC 대시보드 설계

결품 데이터 모니터링 뿐만 아니라, 시간대 / 구역별 결품 발생 패턴을 파악할 수 있도록 차트 형태로 시각화하였습니다. 또한, 결품 → 충진까지 소요 시간을 기입하여 병목이 있는 구간을 빠르게 확인할 수 있도록 했습니다.

결과 1

결품 관리의 정량화 달성

기존엔 현장 직원/마트 관리자의 감에 의존하거나, 결제 데이터만으로 의사결정할 수 있었지만, 현재는 결품 데이터 기반 운영으로 전환되어 결품 대응 시간 및 프로세스 개선 가능해졌습니다.

결과 2

서비스 확장 검토

1개 지점 PoC로 시작하여 CCTV 31대로 시작했지만, 비즈니스 가치가 검증되어 3개 지점 추가 도입을 검토 중입니다.
